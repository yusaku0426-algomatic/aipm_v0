## クリティカルパス

```mermaid
graph TD
    A[プロジェクト立ち上げ] --> B[KPI機能実装]
    B --> C[認可機能実装]
    C --> D[人事部WF開発]
    D --> E[本番運用開始]
    E --> F[ROI評価完了]
    F --> G[プロジェクト完了]
```

**クリティカルパス**: プロジェクト立ち上げ → KPI機能実装 → 認可機能実装 → 人事部ワークフロー開発 → 本番運用開始 → ROI評価完了 → プロジェクト完了

**重要な依存関係**:
- 人事部ワークフロー開発は認可機能完了後に開始
- 本番運用は全技術基盤完成後に開始  
- ROI評価は本番運用データ収集後に実施
